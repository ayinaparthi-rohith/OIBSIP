# Task 8: Capture Network Traffic with Wireshark

## Objective

The objective of this task is to capture and analyze network traffic using Wireshark. This task helps in understanding how data travels across a network, how communication occurs between devices, and how network packets can be inspected for security analysis.

Wireshark is one of the most widely used network protocol analyzers and is commonly used by network administrators, security analysts, incident response teams, and penetration testers for monitoring and troubleshooting network activity.

---

# Tool Used

* Wireshark 4.6.6
* Npcap Driver
* Windows 11 Pro

---

# About Wireshark

Wireshark is an open-source network protocol analyzer that captures and displays network packets in real time. It allows users to inspect packet contents, identify communication patterns, troubleshoot network issues, and analyze security incidents.

Wireshark supports hundreds of protocols and provides detailed information about packet headers, source and destination addresses, communication ports, and protocol-specific data.

---

# Task Execution Steps

## Step 1: Install Wireshark

Wireshark was downloaded and installed on the system.

During installation, the Npcap driver was also installed to enable packet capturing functionality.

### Purpose

Npcap provides low-level access to network interfaces, allowing Wireshark to capture packets directly from the network.

---

## Step 2: Launch Wireshark

After installation, Wireshark was launched successfully.

The application displayed all available network interfaces on the system, including:

* Wi-Fi
* Loopback Adapter
* Local Area Connections

### Purpose

The available interfaces represent different network connections through which data can be captured.

---

## Step 3: Select Active Network Interface

The Wi-Fi interface was selected because it was the active connection used to access the internet.

### Purpose

Selecting the correct interface ensures that Wireshark captures relevant network traffic generated by the system.

---

## Step 4: Start Packet Capture

Packet capture was started by double-clicking the Wi-Fi interface.

Wireshark immediately began displaying live network packets.

The captured traffic included multiple protocols such as:

* TCP
* NBNS
* LLMNR
* TLS
* HTTP

### Purpose

Packet capturing allows real-time monitoring of network communication between devices and servers.

---

## Step 5: Generate Network Traffic

To generate observable traffic, a web browser was used to access websites.

The website used for HTTP analysis was:

```text
http://neverssl.com
```

### Purpose

Most modern websites use HTTPS encryption. The selected website provides unencrypted HTTP traffic, making packet analysis easier.

---

## Step 6: Apply HTTP Filter

To focus only on HTTP communication, the following display filter was applied:

```text
http
```

After applying the filter, Wireshark displayed only HTTP packets.

### Purpose

Filtering reduces unnecessary traffic and allows detailed analysis of specific protocols.

---

## Step 7: Analyze Captured Packets

A captured HTTP packet was selected for detailed inspection.

The packet contained the following information:

| Field                  | Value           |
| ---------------------- | --------------- |
| Source IP Address      | 192.168.1.5     |
| Destination IP Address | 34.223.124.45   |
| Protocol               | HTTP            |
| Request Method         | GET             |
| Website                | neverssl.com    |
| Response Status        | HTTP/1.1 200 OK |

### Packet Analysis

The captured packet represents a client request sent from the local machine to a remote web server.

The GET request indicates that the client requested a web resource from the server.

The server responded with an HTTP 200 OK message, indicating that the request was processed successfully and the requested content was delivered.

---

# Protocol Analysis

## HTTP (Hypertext Transfer Protocol)

HTTP is a communication protocol used for transferring web content between clients and servers.

### Observed Requests

```text
GET /
GET /online
GET /favicon.ico
```

### Observed Responses

```text
HTTP/1.1 200 OK
HTTP/1.1 301 Moved Permanently
```

### Explanation

#### GET Request

The GET method is used by a client to request resources from a web server.

#### HTTP 200 OK

This response indicates that the server successfully processed the request and returned the requested content.

#### HTTP 301 Moved Permanently

This response indicates that the requested resource has been permanently moved to a different location.

# Security Observations

During packet analysis, the following observations were made:

### Plain Text Communication

HTTP traffic is transmitted without encryption.

This means that anyone monitoring the network may be able to view transmitted data.

### Visibility of Requests

The requested URLs and communication details were visible within the packet contents.

### Importance of HTTPS

Modern websites generally use HTTPS to protect communication through encryption.

HTTPS prevents attackers from easily reading intercepted network traffic.

# Limitations

### Encrypted Traffic

HTTPS traffic cannot be fully inspected without proper decryption methods.

### Large Amounts of Data

Busy networks may generate thousands of packets, making analysis time-consuming.

### Privacy Considerations

Packet captures may contain sensitive information and should be handled responsibly.
